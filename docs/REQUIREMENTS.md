# 요청사항
* 모든 요청사항은 docs 폴더 내의 spec documents(구조 문서)에 먼저 업데이트한 후, 이를 기반으로 체계적이고 경제적으로 코드 패치를 수행하는 것을 원칙으로 한다.

## REQUIREMENTS 01
1. 
[문제] 현재 시각적으로 락커 영역 사이가 곡선으로 이어져 있지 않아보이며, 두 락커 영역과 떨어진 엉뚱한 곳이 곡선들이 위치하고 있습니다.
[해결] 락커의 한 변과 다른 락커의 한 변이 확실히 곡선과 그 패스를 따라가는 텍스트로 이어질 수 있도록 오류 문제를 파악 및 패치해주세요.

2.
[현상황] 락커 영역을 클릭하면 내부에 새로운 색의 배경이 확산되며 락커 영역 내부에 텍스트 콘텐츠가 보입니다.
[해결] 락커 영역에 마우스 커서를 정지된 상태로 오래 홀드하면 새로운 색의 배경이 확산되고, 이것이 락커 영역 밖까지 나가며 화면 전체를 채우도록 패치합니다. 즉, 콘텐츠 영역이 더 넓을 수 있도록 합니다.

## REQUIREMETNS 02
1. 
[문제] (1) 여러 museum node가 연결된 경우, 중앙이 자연스럽게 이어지지 못하고 중앙의 원 모양이 두드러집니다.
(2) 여러 museum node가 연결되었을 때 한 노드로 향하는 가지를 hover하면 그 가지만 부자연스럽게 끊어진 형태로 색상이 바뀝니다. 이는 의도한 디자인이 아닙니다.
[해결방안 제안] 중앙에 원 도형을 락커 영역(정사각형)을 45도 회전한 모양과 동일한 마름모꼴로 변경합니다. 락커의 한 변과 마름모꼴의 한 변과 연결하는 노드 선을 그립니다.

* 가능 여부 및 구현 방향을 먼저 이 문장 아래에 제안 작성한 이후, 승인되면 실제 코드 패치를 진행해주세요.

[제안]
* **가능 여부**: 가능함
* **구현 방향**:
    1. `components/RelationsLayer.tsx` 내의 허브 렌더링 로직을 수정합니다.
    2. **도형 변경**: 기존 `circle` 기반의 허브를 `path` 또는 회전된 `rect`를 사용하여 마름모(45도 회전된 정사각형) 형태로 변경합니다. 크기는 락커(`TILE_WIDTH`)와 시각적 균형을 맞춥니다.
    3. **연결점 계산**: 멤버와 연결되는 허브 상의 좌표(`hubA`, `hubB`)를 원의 둘레가 아닌 마름모의 둘레(`|x| + |y| = R`) 상의 점으로 계산하도록 수식을 변경합니다.
    4. **경로 생성**: `mergedPath` 생성 시 두 연결점 사이를 잇는 구간을 원호(Arc)가 아닌 직선(Line)으로 변경하고, 마름모의 모서리를 포함해야 하는 경우 해당 모서리 좌표를 경로에 추가합니다.
    5. **Hover 효과 개선**: 리본(Ribbon)이 마름모의 변에 빈틈없이 밀착되도록 하여, hover 시 색상 변화가 자연스럽게 이어지도록 처리합니다.
